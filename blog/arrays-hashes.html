Arrays and Hashes in Ruby

February 10th, 2016
By Eric Gumerlock

It can sometimes be confusing in Ruby when you are thinking of organizing data, whether to use arrays or hashes. Both are useful, but just for different applications. Lets begin by defining both and array and a hash. An array is a set of data created by setting a variable equal to a bracket of data separated by commas, like this my_array = [“my”, “job”, “fun”]. Each of those strings is a value, and each is automatically indexed by integers, starting at 0, and increasing by 1. So “my” has the key 0, “job” has the key 1, and “fun” has the key 2. These can be called by writing my_array[1] which will return “job”. Each key can only have one value, so if you attempt to overwrite one of the pieces of data, say my_array[1] = “going home”, Ruby will overwrite “job” with “going home”.

A hash is like a next level array. You can customize the key of each value, with a string or symbol. Also, a key can contain a number of elements, such as an array. Lets get into a few commands before the theory gets too confusing. You make a hash like so my_hash = {“key” => “value”, “John” => “Smith”} This would create a key of “John”, with value “Smith” and a key of “key” with value “value”. There are also other ways to create a hash, such as my_hash = Hash.new and later you are able to add key/value pairs with this notation my_hash[“John”] = “Smith”. And to call that value, its simply my_hash[“John”] to see what its value is.

So when do we use arrays and hashes? Well if you have a data set, say a bunch of numbers that are related in some way, you probably want to use an array. If each of those numbers corresponds to a person, like age for example, we would rather make a hash since you can index the ages by name, using names as keys and ages as values. Basically, if your data needs to be labeled using a string or a symbol, you want to use a hash. But if you are dealing with only numerical values, an array is more appropriate. An array is also good for holding a lot of random data, numbers, strings, etc., but with a very simplistic index of integers. Sometimes this is great for organizing random and unorganized data first.
