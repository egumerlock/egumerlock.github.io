Ruby Enumerable Methods: group_by

February 19th, 2016
By Eric Gumerlock

There are number of very useful Enumerable methods in Ruby one of which is pretty useful called .group_by. It takes an array or also a group of numbers (1..6) and groups the elements of that array by a block. If no block is given it just returns an enumerator instead. But you usually won’t be using that way, and you will be passing a block most of the time. So lets talk about what can go in the block, and the different outcomes you will receive. The first and most intuitive way to use .group_by would be to just .group_by { |e| e } which would return a hash that groups identical elements into values. The keys would be identical to the value/values of that particular element. Another way we could use this method, say with an array of strings, would be like this .group_by { |e| e[0] } which would sort by the first letter of the string, and group the values that start with that letter into keys corresponding to that first letter. Here’s an example: 

      birds = ["Golden Eagle", "Gyrfalcon", "American Robin",
      "Mountain BlueBird", "Mountain-Hawk Eagle"]
      grouped_by_first_letter = birds.group_by { |s| s[0] }

      # {G=>[Golden Eagle, Gyrfalcon], A=>[American Robin],
      # M=>[Mountain BlueBird, Mountain-Hawk Eagle]}
You can see how this method can be very useful. If we use methods within .group_by such as .length, we will end up with a hash with keys of the length, and values of the elements with that length. As we become more familiar with .group_by, we can start including a case within the block like so: 

      raptors = ["Golden Eagle", "Bald Eagle", "Prairie Falcon", "Peregrine Falcon",
      "Harpy Eagle", "Red Kite"]

      kinds = raptors.group_by do |species|
      case species
      when /.*Eagle.*/
      Eagle
      when /.*Falcon.*/
      Falcon
      else
      unknown
      end
      end

      # {Eagle=>[Golden Eagle, Bald Eagle, Harpy Eagle],
      # Falcon=>[Prairie Falcon, Peregrine Falcon], unknown=>[Red Kite]}
Lastly, we can use .group_by with a numerical function, which then groups the array elements into keys, and their corresponding solutions to those functions. For example: 

      (1..6).group_by {|i| i%3}   #=> {0=>[3, 6], 1=>[1, 4], 2=>[2, 5]}
This method seems like it takes awhile to master all the ways that it can be used, but once you get a hang for what it can do, it becomes a useful tool in your Ruby toolbelt.
