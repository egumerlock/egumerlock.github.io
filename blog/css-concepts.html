Using Inline vs. Inline-block

Inline and inline-block can be two confusing ways to display elements in HTML using CSS. I ran into some trouble today while programming and decided to write a small explanation to help those who really want to get the bottom of which one to use and when. Each type of element has specific attributes that make them handy if you know them. Inline elements respect left and right margins and padding, yet they do not have top and bottom margin and padding qualities. They cannot have a width and height set in their attributes, but they allow other elements to sit to their left and to their right. Inline-block elements allow other elements to sit to the left and to the right, AND respect top & bottom margins and padding attributes. Also, inline-block elements respect and will change with height and width formatting, which makes inline-block a great choice for more freedom of formatting. Realistcally, you could only use inline-blocks, and then make sure all formatting is the same for all elements within the line, and you will be able to make elements that look and perform exactly the same as inline elements. However, if you know you want your elements to look exactly like inline elements, go ahead and make them inline. Inline-block will, more often than not, make more send to use.
